记录使用 Emacs 搭配 Hugo 搭建静态博客的过程。预备条件：创建自己的 GitHub Page 仓库。

Hugo 准备

安装 Hugo \\

Hugo 是用 Go 语言编写的静态站点引擎，除了手动编译、系统软件源安装外, 还可以下载二进制 release 文件安装。

生成站点

安装以后，我们需要建立一个博客根目录，存放博客的配置以及博文等。

1

hugo new site /path/to/site

比如选择主目录下 site 文件夹为博客根目录。

1

hugo new site ~/site

执行生成站点命令后，博客根目录下有这些文件夹和文件，

新建博文

在 hugo 的文件结构中， content 目录下每一个一级子目录都称为 section, 通常我们会将我们的博文放到名为 post 的 section 中，运行命令

1

hugo new post/first.md

会自动在 ~/site/content/post/ 目录下生成文件 first.md. 然后编辑该 markdown 文件内容即可，它的内容会作为博客页面的内容。同时 post 目录下的 markdown 文件默认会在博客的首页展示。

安装主题

因为 hugo 默认没有自带的主题，为了预览生成的博文，还需要安装主题。可以网上搜索 hugo 主题，选择自己喜欢的主题，下面以 even 主题为例，介绍安装流程。

  1. 将主题下载到博客根目录的 themes 文件夹内

    1
2

    cd themes
git clone https://github.com/olOwOlo/hugo-theme-even even

  2. 编辑根目录 config.toml，设置主题为 even:

    1

    theme = "even"

    并参考仓库中的 exampleSite 目录中的 config.toml 文件修改站点配置文件。

部署到 github pages

接下来将本地的博客部署到 github pages 上。根据官方演示创建仓库，添加 index.html 即可。

有了自己的 github page, 在博客站点根目录执行

1

hugo --theme=hyde --baseUrl="http://name.github.io/"

如果一切顺利，所有静态页面都会生成到 public 目录，接下来将 pubilc 目录里所有文件 push 到 github page 仓库，

1
2
3
4

cd public
git init
git remote add origin https://github.com/name/name.github.io.git
git add -A
git commit -m "first commit"
git push -u origin master

#!/bin/bash
# file deploy.sh
echo -e "\033[0;32mDeploying updates to GitHub...\033[0m"

# build the project
hugo -t even

cd public
git add .
msg="rebuilding site `date`"

if [ $# -eq 1 ]
  then msg="$1"
fi

git commit -m "$msg"

# push source to github
git push upstream master

# come back to blog root
cd ..

之后，即可运行 ./deploy.sh 就可以将博客更新并部署到 github 了。

Emacs 端设置

安装 ox-hugo

ox-hugo 可通过 melpa 获取，安装后添加下面的配置

1
2
3

(with-eval-after-load 'ox
  (require 'ox-hugo)
  (setq org-hugo-default-section-directory "post"))

ox-hugo 相关快捷键

下面的两个快捷键用于快速设置子标题属性，比如若每个 org subtree 对应一个博文，有必要设置 EXPORT_FILE_NAME 属性，这时通过 C-c C-x p file name RET <导出时的博文标题> RET 可以快速对当前子标题添加下面内容

1
2
3

:PROPERTIES:
:EXPORT_FILE_NAME: <导出时的博文标题>
:END:
desc org command remark
C-c C-x p org-set-property quickly set property for each subtree
C-c C-q org-set-tags-command set headding tags (the blog’s categories or tags)

若光标位于 Org 标题上，可以用 C-c C-c 替代 C-c C-q.

org-capture 快速添加博文标题

;; Populates only the EXPORT_FILE_NAME property in the inserted headline.
(with-eval-after-load 'org-capture
  (defun org-hugo-new-subtree-post-capture-template ()
    "Returns `org-capture' template string for new Hugo post.
See `org-capture-templates' for more information."
    (let* ((title (read-from-minibuffer "输入博客标题: ")) ;Prompt to enter the post title
           (fname (org-hugo-slug title)))
      (mapconcat #'identity
                 `(
                   ,(concat "* TODO " title)
                   ":PROPERTIES:"
                   ,(concat ":EXPORT_FILE_NAME: " fname)
                   ":END:"
                   "%?\n")          ;Place the cursor here finally
                 "\n")))

  (add-to-list 'org-capture-templates
               '("h"                ;`org-capture' binding + h
                 "create a Hugo post"
                 entry
                 ;; It is assumed that below file is present in `org-directory'
                 (file "site.org")
                 (function org-hugo-new-subtree-post-capture-template))))

ox-hugo 设置博文作者

参考：

  • https://ox-hugo.scripter.co/doc/author/#tweaking-your-hugo-template-to-support-list-author
  • https://ox-hugo.scripter.co/doc/author/#forcing-author-to-be-a-string--alternative

对于一些主题(比如 even 主题)，ox-hugo 导出的 markdown 文件 front matter 内 author 部分无法解析。 执行 hugo server 时提示会渲染报错。

为了解决这个问题，可以强制 ox-hugo 导出作者名为一个字符串，而不是列表。对 整个 Org 文件 设置 两个 属性： author 、 hugo_custom_front_matter, 注意两者缺一不可。

1
2

#+author:
#+hugo_custom_front_matter: :author "ding"

另一方面，官方也给出了 org subtree -> one post 对应的作者属性设置方法：

1
2
3

:PROPERTIES:
:EXPORT_AUTHOR: Firstname LastName
:END:

但个人使用 even 主题过程中，该方法无法跳过 hugo server 不可渲染博文作者的 bug. 所以暂时不考虑给每个 org subtree 设置不同作者的流程。
