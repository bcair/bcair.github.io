1 安装 hugo 到系统并生成站点

1.1 新建 hugo 博文

Hugo 是用 Go 语言编写的静态站点引擎，除了手动编译、系统软件源安装外, 可以下载二进制 release 文件安装。

安装以后，我们需要建立一个博客根目录，存放博客的配置以及博文等。

1

hugo new site /path/to/site

比如选择主目录下 site 文件夹为博客根目录。

1

hugo new site ~/site

执行生成站点命令后，博客根目录下有这些文件夹和文件，

1.2 添加博文内容

在 hugo 的文件结构中， content 目录下每一个一级子目录都称为 section, 命令

1

hugo new <SECTIONNAME>/<FILENAME>.<FORMAT>

会新建 markdown 文件。进入博客站点根目录后运行

1

hugo new post/first.md

会新建 section post 以及新的文件 /path/to/site/content/post/first.md.

与 xo-hugo 配合，默认们将我们的博文放到 post section 中，运行命令

1

hugo new post/our_blog.md

编辑 our_blog.md 文件内容即编辑我们的博文。=post= 目录下的 markdown 文件默认会在博客的首页展示。

1.3 安装主题

hugo 框架下博文都是 markdown 文件，为了在网页中浏览博客，还需要安装hugo主题。可以网上搜索 hugo 主题，选择自己喜欢的主题，下面以 even 主题为例，介绍安装流程。

首先将主题下载到博客根目录的 themes 文件夹内

1
2

cd themes
git clone https://github.com/olOwOlo/hugo-theme-even even

接着编辑根目录 config.toml，添加一行 theme = “even"; 然后并参考 even git 仓库 exampleSite 目录中的 config.toml 文件修改站点配置文件 /root/of/hugosite/config.toml.

(unless (package-installed-p 'ox-hugo)
  (package-install 'ox-hugo))

(with-eval-after-load 'ox
  (require 'ox-hugo)
  (setq org-hugo-export-with-section-numbers 3
        org-hugo-export-with-toc t
        org-hugo-footer ""
        org-hugo-front-matter-format "toml"
        org-hugo-section "post"
        ;; ~C-a~ will translate to <kbd>C-s</kbd>
        org-hugo-use-code-for-kbd t))

这里的 org-hugo-section 的值与 hugo 博客根目录下 content/post section 对应。 M-x customize-group org-export-hugo 可以设置相关的其他变量。

2.2 ox-hugo 如何管理 Org 文档？

有效的 ox-hugo org 文档或子树有两类：

有效的 Hugo post subtree
添加 EXPORT_FILE_NAME 属性的 Org 子树。
有效的 Hugo post org file
添加 #+TITLE 关键字的 org 文档，且 org 文档不含有效的 hugo post 子树

总结就是有效的 subtree 和 file 都设定了转化后的 title front matter.

ox-hugo 将有效的 Org 文件或者 Org 子树转为 /hugo/site/content/post/ 目录下的一篇 markdown 文档。

  • ox-hugo 负责 Org 向 hugo markdown 的转化
  • hugo 负责博客站点管理的 markdown 文件的渲染

快捷键：

export object shortcuts scope 1 scope 2
one subtree or entire file C-c C-e H H current subtree file
all subtrees or entire file C-c C-e H A all subtrees file
only current file C-c C-e H h file

表中 scope 1 优先级高于 scope 2, 一旦优先级高的区间为有效的 ox-hugo org 单元，会转化区间内容会 hugo markdown 文件, 不会进一步转化低优先级的区间。

2.3 编辑 Org 模式子树的快捷键

desc org command remark
C-c C-x p org-set-property quickly set property for each subtree
C-c C-q org-set-tags-command set headding tags (the blog’s categories or tags)
C-c C-s org-schedule Insert a “SCHEDULED” string with a timestamp

执行 C-c C-x p export-file-name blog_title 可以快速将当前子标题设置位有效的 hugo-org-subtree, 同时设置导出后的博文标题为 blog_title.

快捷键 C-c C-q 可以在 Org 标题上设置标签。设置博文的标签和种类等同于设置 markdown front matter 中的 tags categories. 若光标位于 Org 标题上，可以用 C-c C-c 替代 C-c C-q.

快捷键 C-c C-s 可以在 Org 标题下方添加一行如

1

SCHEDULED: <2020-10-04 Sun>

的内容，ox-hugo 将该行转化为 hugo front matter PublishDate.

;; Populates only the EXPORT_FILE_NAME property in the inserted headline.
(with-eval-after-load 'org-capture
  (defun org-hugo-new-subtree-post-capture-template ()
    "Returns `org-capture' template string for new Hugo post.
See `org-capture-templates' for more information."
    (let* ((title (read-from-minibuffer "输入博客标题: ")) ;Prompt to enter the post title
           (fname (org-hugo-slug title)))
      (mapconcat #'identity
                 `(
                   ,(concat "* TODO " title)
                   ":PROPERTIES:"
                   ,(concat ":EXPORT_FILE_NAME: " fname)
                   ":END:"
                   "%?\n")          ;Place the cursor here finally
                 "\n")))

  (add-to-list 'org-capture-templates
               '("h"                ;`org-capture' binding + h
                 "create a Hugo post"
                 entry
                 ;; It is assumed that below file is present in `org-directory'
                 (file "site.org")
                 (function org-hugo-new-subtree-post-capture-template))))

3 hugo server 无法在 even 主题下渲染 ox-hugo 导出的 markdown 文件

参考：

  • https://ox-hugo.scripter.co/doc/author/#tweaking-your-hugo-template-to-support-list-author
  • https://ox-hugo.scripter.co/doc/author/#forcing-author-to-be-a-string--alternative

对于一些主题(比如 even 主题)，hugo server 无法渲染成功，原因是 ox-hugo 默认导出的 author front matter 参数为 TOML/YAML 列表格式，这些主题不支持该类型。

为了解决该问题，需要

  1. 使默认的 author 为 nil

    • file post: #+author: 设为空
    • subtree post: 属性 :EXPORT_AUTHOR: 设为空

  2. 强制 ox-hugo 导出时将作者名称转为字符串形式（而不是列表），不论是 file post 还是 subtree post, 都需要添加关键字指定作者名

    1

    #+hugo_custom_front_matter: :author "vailete"

4.2 subtree post 设定

一个有效的 subtree post 至少有5个 org 元数据，标记在下表中的 required 字段，比 file post 多了一个必须属性 :EXPORT_FILE_NAME: exported-markdown-file-name, 该属性不可为空。

org meta desc hugo front matter required remark
#+hugo_base_dir keyword Y
#+hugo_custom_front_matter: author "name" keyword author Y some theme require it.
* blog title heading title, draft = false Y with no TODO keyword, set draft = false
:EXPORT_FILE_NAME: a property Y ox-hugo needs it mark one subtree as effective
:EXPORT_AUTHOR: property Y disable default ox-hugo action for “author”
* TODO blog title TODO keyword & heading title, draft = true
* DONE blog title TODO keyword & heading title, draft = false
CLOSED: [2017-09-11 Mon 14:32] auto inserted with DONE state date
:EXPORT_DATE: 2020-10-04 property date
SCHEDULED: <2020-10-04 Sun> C-c C-s generate PublishDate
:EXPORT_HUGO_AUTO_SET_LASTMOD: t property lastmod

再次强调，属性 EXPORT_FILE_NAME 非空才会被 ox-hugo 认为是有效的 subtree post, 但该属性值

  1. 不会作为导出后的博文标题
  2. 作为 subree post 对应的 markdown 文件名.

例子

#+hugo_base_dir: ~/site
#+hugo_custom_front_matter: :author "vailete"

* TODO blog title                       :@name:tag1:tag2:
  SCHEDULED: <2020-10-04 Sun>
  :PROPERTIES:
  :EXPORT_AUTHOR: (blank)
  :EXPORT_FILE_NAME: exported-markdown-file-name
  :EXPORT_HUGO_AUTO_SET_LASTMOD: t
  :END:

subtree post 的 categories front matter 有3种方法可以设置:

  1. 属性 EXPORT_HUGO_CATEGORIES, 拥有最高优先级，覆盖其他如关键字
  2. 关键字 #+hugo_categories: x y
  3. 标题 tag 添加前缀 “@” ** level 2 heading :@tools:@animal:

标题前缀为 @ 的标签，如当前标题有标签 :work:@simple:, 表示 subtree post 的 hugo front matter categories 列表中应包含 simple.

更详细的设定请看 官方文档

4.3 file post vs subtree post

This works but you won’t be able to leverage Org-specific benefits like tag and property inheritance, use of TODO states to translate to post draft state, auto weight calculation for pages, taxonomies and menu items, etc.

虽然 file post 的关键字添加比较快捷，但 subtree post 有下面的优点

  1. C-c C-q 设定 post 的标签、属性，以及各 org subtree 可以实现对它们的继承
  2. C-c C-t 设定 TODO 关键字，由此可以自动设置 front matter 中的 draft pulishdate
