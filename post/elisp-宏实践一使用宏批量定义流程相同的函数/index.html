函数功能：快速访问 django 项目视图、表单模块和模板文件

之前为了方便 django 项目跳转视图、表单模块，写了几个函数，快捷键调用时可以迅速给出对应的文件列表便于打开。

主要的函数和快捷键(evil normal state map)如下：

function name binding in normal state
my/django-find-view ,dv
my/django-find-form ,df
my/django-find-template ,dt
my/django-find-static ,ds

如此，连续按下 , d v/d/t/s 三个键即可访问任意统一应用下的视图、表单、模板等。

主要函数的定义见下文，写出三个函数后，发现长得像四胞胎。。。然后试着编写一个宏来批量定义出这几个函数，不过新手难免对宏的编写理解不到位，自己尝试的还是循环体中调用宏这种“高级”操作，怎么都无法展开为想要的函数，一度怀疑宏的存在有什么意义。加上写出来的函数已经满足预期目的，就将这项任务加入到代办事项中吃灰了。

最近，翻写一堆 leader-map 时，尝试着用宏折腾了一下，发现之前忽视的几个问题。

首先，普通函数的参数默认先求 symbol 对应的值并传递给函数，使用 quote 来表示

字面量 函数参数含义 宏参数含义
‘ab symbol cons (quote (cons ab . nil))
ab the value of symbol symbol 'ab
  1. 同样的 symbol 样式在函数调用中的 symbol 样式的参数在宏定义时为 cons 类型
  2. 函数中 variable 样式参数

宏看了下文档，理解了循环体中使用宏的难度，

与宏相关的代码有两个运行时：展开时、调用时。 有时可能宏展开没有错，但展开后的函数执行有错。无法分清错误发生在 展开时 还是 调用时 时，往往容易将

  1. 函数接受错误的参数这种函数调用错误
  2. 与宏展开错误

所混淆。比如

(defun my/django--get-django-root ()
  "return valid django root absolute path, default is minisaas project.
with prefix, select root"
  (if (project-current)
      (if (file-exists-p
           (expand-file-name "manage.py" (project-root (project-current))))
          (cons t                       ; is a django root
                (expand-file-name (project-root (project-current))))
        (cons nil                    ; origin place not in a django project
              (if current-prefix-arg
                  (read-directory-name "select Django root: " "~/projects/")
                (expand-file-name "~/projects/minisaas/"))))))

(defun my/django--apps ()
  "Return a list whose car is root path and other are app name"
  (let* ((root-info (my/django--get-django-root))
         (root (cdr root-info))
         (app-dirs (seq-filter
                    (lambda (subdir)
                      (and
                       ;; 1. is directory
                       (file-directory-p subdir)
                       ;; 2. excluding "." ".." ".git"
                       (not (member (file-name-nondirectory subdir)
                                    '("." ".." ".git" "__pycache__" "venv")))
                       (file-exists-p (expand-file-name "apps.py" subdir))))
                    (directory-files (expand-file-name root) t)))
         (app-name-list (seq-map
                         (lambda (x)
                           (file-name-nondirectory (directory-file-name x)))
                         app-dirs)))
    (cons root-info app-name-list)))

(defun my/django--cur-app ()
  "return current app path"
  (let* ((info (my/django--apps))
         (django-p (caar info))
         (root-path (cdar info))
         (app-path-list (seq-map (lambda (app-name)
                                   (expand-file-name app-name root-path))
                                 (cdr info)))
         (cur-dir (if django-p
                      ;; in a django project
                      (cond ((eq major-mode 'dired-mode)
                             (dired-current-directory))
                            (t
                             (file-name-directory (buffer-file-name))))
                    ;; otherwise
                    (read-directory-name "select APP: " root-path)))
         (cur-app
          (seq-find (lambda (app-path)
                      (file-in-directory-p cur-dir app-path))
                    app-path-list)))
    cur-app))

(defun my/django-find-view ()
  (interactive)
  (let* ((app-path (my/django--cur-app))
         (app-name (file-name-nondirectory (directory-file-name app-path)))
         (prompt (format "[%s] view: " app-name))
         (view-dir (expand-file-name "views/" app-path))
         (view-files (seq-filter (lambda (x)
                                   (and (equal "py" (file-name-extension x))
                                        (not (member x '("__init__.py")))))
                                 (directory-files view-dir))))
    (find-file (expand-file-name (completing-read prompt view-files)
                                 view-dir))))

(defun my/django-find-template ()
  (interactive)
  (let* ((app-path (my/django--cur-app))
         (app-name (file-name-nondirectory (directory-file-name app-path)))
         (prompt (format "[%s] HTML template file: " app-name))
         (template-dir (expand-file-name (concat "templates/" app-name) app-path))
         (template-dir (if (file-directory-p template-dir)
                           template-dir
                         (expand-file-name "templates/" app-path)))
         (template-files (seq-filter (lambda (x)
                                       (and (equal "html" (file-name-extension x))
                                            t))
                                     (directory-files template-dir))))
    (find-file (expand-file-name
                (completing-read prompt template-files)
                template-dir))))

给三个函数定义快捷键后，就可以使用了。

三个函数的思路基本一致：

  1. 确定应用路径，名称
  2. 找到视图、表单、模板保存的目录
  3. 过滤对应目录中的文件，比如访问模板文件时，过滤掉后缀不是 html 的文件
  4. 调用 find-file 和 completing-read, 传入过滤后的文件列表，实现文件补全

差异只在于保存的路径随着访问文件的种类：视图、模板、表单，有些差异；以及过滤规则不一样。

下面首先定义一个宏处理不同的过滤规则，然后用另一个宏来实现补全，将三个函数的定义结合到一起。

宏1: 根据访问文件类型过滤文件列表

这里的过滤很简单：函数 my/django-find-view 只列出 python 模块, my/django-find-template 只显示 HTML 文件。

命名宏 my/django--file-filter 执行过滤判断。实现目标：

  1. 传递字符串参数 "py", 宏展开代码过滤 python 文件
  2. 传递字符串参数 "html", 展开后的代码过滤 html 文件

(defmacro my/django--file-filter-v0 (extension)
  (cond ((string= extension "py")
         `(lambda (x)
            (and (equal "py" (file-name-extension x))
                 (not (member x '("__init__.py"))))))
        ((string= extension "html")
         `(lambda (x)
            (equal "html" (file-name-extension x))))))

实际就是将对应的函数形式稍微修改为 macro 形式。

(defmacro m/django--file-filter (extension)
  `(lambda (x)
     "fileter files by extension or maybe something else"
     (and (equal ,extension (file-name-extension x))
          ,(if (equal extension "py")
               '(not (member x (list "__init__.py")))
             t))))

(defvar django-map-alist '(("view" . "py") ("form" . "py") ("template" . "html")))

(defmacro m/django-find (type) ; type: "view" "form" "template"
  `(defun ,(intern (concat "my/django-find-" type)) ()
     (interactive)
     (let* ((app-path (my/django--cur-app))
            (app-name (file-name-nondirectory (directory-file-name app-path)))
            (prompt (format (concat "[APP %s] " ,type ": ") app-name))
            (target-dir (seq-find 'file-directory-p
                                  (list
                                   (expand-file-name (concat ,type "s/"  app-name) app-path)
                                   (expand-file-name (concat ,type "s/") app-path))))
            (target-files (seq-filter
                           (m/django--file-filter
                            ,(alist-get type django-map-alist nil nil #'equal))
                           (directory-files target-dir))))
       (find-file (expand-file-name (completing-read prompt target-files)
                                    target-dir)))))

该宏统一了访问视图、表单等的流程，不再需要重复大量的代码来定义 上文 的三个函数，只要下面三行语句即可实现相同的结果:

小结

因为 lisp 中 symbol 有函数定义、符号名字符串、变量值、属性列表四个属性空间，宏的定义通常混用前三个特性，侧重处理 symbol 字面量的处理，所以宏的编写相比函数比较困难。难点在于：

  1. 宏的参数作 symbol 处理，而函数的参数为求值
  2. 要考虑展开时和调用时，从而决定哪部分被 backquoted

以下面的宏为例

1
2
3
4
5
6

(defmacro m/consp (x)
  (if (consp x)
      "yes"
    "not"))

(macroexpand-1 '(m/consp 'ab))
;; => "yes"
(macroexpand-1 '(m/consp ab))
;; => "not"

可见宏对 'ab 的处理与函数对其的处理大相径庭。

其他语法注意点: 宏 django-find 使用 intern 而不是 make-symbol, 因为后者不会全局定义(intern)一个函数。
