HTML 表单发送文件

由于HTTP是一种文本协议，所以处理文件这样的二进制数据(或者大家都这样认为的) 会有一些特殊的要求，而表单中其他数据都是文本数据。

表单属性 enctype

该属性指定提交表单时，生成的请求头中 Content-Type 的值。该数据头非常重要，因为它告诉服务器正在发送的数据类型。

enctype 为 encoding type 的简写，即编码类型。

该属性的默认值为 application/x-www-form-urlencoded, 意思是：“这是已编码为 URL参数的表单数据”，所以往往我们在服务器中接受到的表单请求中 request.body 内容为 b'a=1&b=2 这种形式。

发送文件的3个步骤

  1. 将 method 属性设置为 POST, 因为文件内容不能放入 URL 参数中；
  2. 将 enctype 值设置为 multipart/form-data, 因为数据将被分为多个部分，每个文件单独占用一个部分，表单正文的文本数据也要占用一个部分；
  3. 包含一个或多个允许用户选择上传文件的表单部件，如 <input type="file">.

示例如下：

<form method="post" enctype="multipart/form-data">
  <div>
    <label for="file">Choose a file</label>
    <input type="file" id="file" name="myFile" accept="image/*" multiple>
  </div>
  <div>
    <button>Send the file</button>
  </div>
</form>

其中，input 元素 accept 属性指定只上传图片文件，点击 <button type="button">"Send the file"</button> 按钮后打开的界面中只显示图片文件和目录以选择，而忽略其它类型的文件。 multiple 属性则允许同时选择多个上传文件，该属性对于某些浏览器可能不兼容。
